# 晚安睡眠管理

声明：本插件90%由Codex-5.5生成，本插件90%由Codex-5.5生成，本插件90%由Codex-5.5生成

这个插件会监听 Bot 自己发出的消息：当 Bot 在夜间主动说出“我睡了”“该睡了”“大家晚安”等短句时，自动进入临时睡眠状态

它也会识别用户让 Bot 睡觉的消息：

- 如果当前不在允许入睡时间内，默认会委婉回复并拦截这条消息，避免进入 Planner
- 如果当前在允许入睡时间内，只记录一次短暂的“待确认入睡”状态，然后放行给主链路判断，并给 Planner 注入一条上下文，说明当前已经符合插件配置的入睡时间
- 只有当 Bot 随后自己发出“晚安”“安安”“大家晚安”“我先睡了”等确认消息后，才会真正进入睡眠

睡眠期间会暂停：

- 新入站消息进入主处理链路
- 表达学习提取和写入
- Planner 响应结果和工具调用
- 睡眠后的后续出站消息

睡眠状态会默认持久化到 `data/plugins/goodnight_sleep_manager/sleep_state.json`。如果主程序、插件 Runner 或插件本身在睡眠期间重启，插件加载时会恢复尚未过期的睡眠状态；如果预计醒来时间已经过去，会自动清理状态文件。手动 `/sleep_wake` 或自然到点唤醒也会清理持久化状态。

为了避免别人诱导 Bot 睡觉，插件默认只接受很短的自我睡眠或群体收尾短句。单独一句“晚安”只有在前面有人于合适时间催睡、且 Bot 自己选择这样回复时才会触发；带 `@`、明显称呼别人、或像“晚安，某某”这种对个人说晚安的内容不会触发

默认正则只提供每类一条参考规则，更多触发词、排除词和催睡表达建议按实际群聊习惯在插件配置里自行添加

## 分群作息

全局“作息”页仍然作为默认时间配置。需要某个群单独使用不同作息时，可以在“分群作息”里添加一条群作息覆盖，填入群号和这组时间：

- `允许入睡开始`
- `允许入睡结束`
- `目标醒来时间`
- `最短睡眠分钟`
- `最长睡眠分钟`
- `醒来随机浮动`

当消息来自该群时，分群作息的优先级高于全局作息；未命中任何群号时继续使用全局作息。用户在允许时间内催睡时，Planner 提示里也会说明当前使用的是分群配置还是全局配置

## 命令

- `/sleep_status`：查看当前睡眠状态
- `/sleep_wake`：手动唤醒

## 注意

当前 MaiBot 的 `maisaka.planner.before_request` hook 不允许直接中止请求，所以插件会在睡眠期间清空 Planner 工具、压缩请求内容，并在响应后清空结果。新消息入口会被拦截，因此正常情况下不会再由新消息触发 Planner

当用户在允许入睡时间内催睡时，插件不会强制 Bot 立刻睡觉；它只会提醒 Planner：当前时间已经落在配置窗口内，不应该再用“太早”“还没到睡觉时间”作为拒绝理由。是否说晚安入睡仍由 Bot 根据上下文自己决定
